# Step 3: 优先级排序

## Goal

对 In Scope 功能进行优先级排序，明确核心场景和验收标准。

## Mandatory Execution Rules

1. 对每个 In Scope 功能分配优先级 (P1/P2/P3)
2. 记录每个功能的核心场景
3. 记录验收标准
4. 更新 STATE.md 的 steps_completed

## Instructions

### 3.1 优先级定义

向用户解释优先级含义:
- **P1 (核心功能)**: 本次必须实现，影响主流程
- **P2 (重要功能)**: 本次应该实现，提升用户体验
- **P3 (增强功能)**: 本次可以实现，锦上添花

### 3.2 优先级分配

对每个 In Scope 功能询问:
"这个功能 [功能名] 的优先级应该是 P1/P2/P3？"

更新功能列表:
| 功能编号 | 功能名称 | 优先级 | 描述 |
|----------|----------|--------|------|
| F-001 | [功能名] | P1 | [描述] |

### 3.3 核心场景定义

对每个 P1/P2 功能询问:
"这个功能的核心使用场景是什么？用户会如何操作？"

记录为:
- **场景编号**: S-001
- **触发条件**: 用户在什么情况下使用
- **操作流程**: 用户的具体操作步骤
- **预期结果**: 用户期望的反馈

### 3.4 验收标准定义

对每个 P1 功能询问:
"这个功能完成的标准是什么？如何判断它已经正确实现？"

记录验收标准:
- **AC编号**: AC-001
- **验收条件**: 具体的可验证条件
- **验证方法**: 如何验证（手动/自动）
