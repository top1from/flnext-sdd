# Step 3: 执行验收测试

## Goal

在 beta 环境上对照需求边界逐一验证功能。

## Instructions

### 3.1 功能验收

按 FR 编号逐项验收:

```
FR-001: {功能描述}
  前置条件: {条件}
  操作步骤:
    1. {步骤}
    2. {步骤}
  预期结果: {结果}
  实际结果: {结果}
  验收状态: ✅ PASS / ❌ FAIL
  备注: {如有问题}
```

### 3.2 非功能验收

- [ ] 页面加载时间 < 3 秒
- [ ] 4 种状态（loading/empty/error/normal）正确展示
- [ ] 浏览器兼容性（Chrome/Edge）
- [ ] 移动端响应式

### 3.3 验收结果

| 结果 | 条件 | 后续 |
|------|------|------|
| **PASS** | 全部 FR 通过 | 生成验收报告，进入发布 |
| **CONDITIONAL_PASS** | 有非阻塞问题 | 记录问题，确认后进入发布 |
| **FAIL** | 有阻塞问题 | 返回开发修复 |

---

**下一步**:
- PASS → ./step-04-report.md（验收报告）
- FAIL → 记录问题 → 返回阶段 4 修复
