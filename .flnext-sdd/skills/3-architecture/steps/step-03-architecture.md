# Step 3: 架构方案设计

## Goal

设计系统架构方案，包括架构图、模块划分、API结构。

## Mandatory Execution Rules

1. 设计系统架构图
2. 划分功能模块
3. 设计API结构
4. 生成架构文档
5. 更新 STATE.md

## Instructions

### 3.1 设计系统架构图

绘制架构层次:

```
┌─────────────────────────────────────────┐
│              前端层                      │
│  [Web/App] ←→ [API Gateway]             │
└─────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────┐
│              服务层                      │
│  [业务服务] ←→ [公共服务]               │
└─────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────┐
│              数据层                      │
│  [数据库] ←→ [缓存] ←→ [存储]           │
└─────────────────────────────────────────┘
```

|----------|----------|----------|------|
| M-001 | [模块名] | [FR列表] | [依赖模块] |

### 3.3 设计API结构

| API编号 | 端点 | 方法 | 功能 | 请求/响应 |
|---------|------|------|------|-----------|
| API-001 | /api/v1/xxx | GET/POST | [功能] | [结构] |

### 3.4 生成架构文档

使用模板生成 docs/sdd/{FEATURE_NAME}/architecture.md。
